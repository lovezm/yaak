# Lessons

- 当用户希望头信息、键值列表更便于复制或更贴近原始文本时，优先使用 `KEY:VALUE` 单行展示，不要默认拆成左右两列。
- 当用户明确说“方便复制”时，不要只把视觉样式改成单行；优先输出为整段纯文本块，避免 `span` 或多元素拼装影响整段选中。
- 当用户要“生成代码”时，优先把结果直接嵌在当前面板可见区域，并提供一键复制，不要额外增加跳转或弹窗步骤。
- 当通过弹窗批量修改当前面板数据时，要确认当前视图会立即刷新；如果内部组件有本地状态，需要在保存后显式触发重建，避免“数据已保存但界面要重载才显示”。
- 如果保存动作依赖异步 `patchModel` 回流，不要在回流前就用旧 props 强制重建组件；优先在中间层维护一个本地即时态，先把用户刚改的内容显示出来。
- 请求头编辑器里的字段展示要贴近真实调试需求：像 `cookie` 这种用户明确要求可见的头值，不要默认按密码字段隐藏。
- 界面文案不要机械地全量中文化；像 `Params` 这类用户明确指定要保留英文的术语，应按用户习惯直接使用英文标签。
- 当参数面板和 URL 输入框表达的是同一份请求地址信息时，优先做即时双向联动，不要让用户改完 `Params` 还要自己 mentally 同步 URL。
